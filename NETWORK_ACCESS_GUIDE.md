## 🔧 不同网络环境配置

### 1. 同一WiFi/局域网（最简单）
**适用场景**：手机和电脑连接同一个WiFi

**访问步骤**：
1. 启动服务器，记下显示的IP地址（如：192.168.1.100）
2. 手机浏览器输入：`http://192.168.1.100:5000`

**可能的问题及解决**：
- 如果无法访问，检查防火墙设置
- 运行：`diagnose_network.bat` 进行诊断

### 2. 不同网络（需要端口转发）
**适用场景**：手机使用移动数据，电脑在家用WiFi

**配置步骤**：

#### 步骤1：配置路由器端口转发
1. 登录路由器管理界面（通常是192.168.1.1或192.168.0.1）
2. 找到"端口转发"或"虚拟服务器"设置
3. 添加规则：
   - 外部端口：5000
   - 内部IP：电脑的局域网IP
   - 内部端口：5000
   - 协议：TCP

#### 步骤2：获取公网IP
```powershell
# 在电脑上执行
curl ifconfig.me
```

#### 步骤3：手机访问
- 使用公网IP访问：`http://你的公网IP:5000`
- 例如：`http://123.456.789.012:5000`

### 3. 使用ngrok（最简单的外网访问）
**适用场景**：不想配置路由器，需要临时外网访问

#### 安装ngrok
1. 访问 https://ngrok.com/
2. 注册并下载ngrok
3. 解压到任意目录

#### 使用ngrok
```powershell
# 启动Flask服务器（先不要关闭）
python app.py

# 新开命令行窗口，启动ngrok
ngrok http 5000
```

#### 访问
- ngrok会提供一个公网地址，如：`http://abc123.ngrok.io`
- 手机使用这个地址访问

## 🛡️ 安全配置建议

### 1. 防火墙配置
```powershell
# Windows防火墙添加规则
netsh advfirewall firewall add rule name="Flask Travel App" dir=in action=allow protocol=TCP localport=5000
```

### 2. 生产环境建议
```python
# 在app.py中添加安全配置
app.config['SESSION_COOKIE_SECURE'] = True  # HTTPS环境
app.config['SESSION_COOKIE_HTTPONLY'] = True
app.config['PERMANENT_SESSION_LIFETIME'] = timedelta(hours=2)
```


## 🎯 最佳实践推荐

### 方案A：家庭/办公室网络
**推荐方法**：同一WiFi + 局域网IP
- 配置简单，无需额外工具
- 适合临时演示或测试

### 方案B：需要外部访问
**推荐方法**：ngrok
- 无需配置路由器
- 提供HTTPS访问
- 适合开发和演示

### 方案C：长期部署
**推荐方法**：云服务器 + 域名
- 专业稳定
- 支持HTTPS
- 适合生产环境

## 🆘 常见问题解决

### 问题1：手机无法访问
**检查清单**：
- [ ] 手机和电脑是否在同一网络（方案A）
- [ ] 防火墙是否允许5000端口
- [ ] 服务器是否正在运行
- [ ] IP地址是否正确

### 问题2：外网无法访问
**检查清单**：
- [ ] 路由器端口转发是否正确
- [ ] 公网IP是否正确
- [ ] ISP是否封锁端口
- [ ] 是否使用ngrok等工具

### 问题3：连接不稳定
**优化建议**：
- 使用有线网络连接电脑
- 关闭不必要的防火墙规则
- 考虑使用云服务器
